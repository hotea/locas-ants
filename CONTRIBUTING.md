# Contributing to Locas-Ants

Thank you for your interest in contributing to Locas-Ants! This document provides guidelines for contributing to the project.

[中文版](#中文版) | [English](#english)

## English

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/locas-ants.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `npm run dev`
6. Commit with clear messages: `git commit -m "feat: add new feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

### Code Style

- Use TypeScript for all new code
- Follow existing code formatting
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Commit Message Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, semicolons, etc.)
- `refactor:` - Code refactoring
- `perf:` - Performance improvements
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

### Areas for Contribution

- **Performance**: Optimize rendering or simulation logic
- **Features**: Add new cell types, behaviors, or UI controls
- **Documentation**: Improve README, add tutorials, or code comments
- **Bug Fixes**: Fix reported issues
- **Internationalization**: Add new language translations
- **Testing**: Add unit or integration tests

### Pull Request Guidelines

- Include a clear description of changes
- Reference related issues if applicable
- Ensure code builds without errors (`npm run build`)
- Test in different browsers if UI changes
- Update documentation if needed

## 中文版

### 开始贡献

1. Fork 本仓库
2. 克隆你的 fork：`git clone https://github.com/yourusername/locas-ants.git`
3. 创建功能分支：`git checkout -b feature/your-feature-name`
4. 进行修改
5. 测试修改：`npm run dev`
6. 提交清晰的消息：`git commit -m "feat: 添加新功能"`
7. 推送到你的 fork：`git push origin feature/your-feature-name`
8. 打开 Pull Request

### 代码风格

- 所有新代码使用 TypeScript
- 遵循现有代码格式
- 使用有意义的变量和函数名
- 为复杂逻辑添加注释
- 保持函数小而专注

### 提交信息规范

我们遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` - 新功能
- `fix:` - 错误修复
- `docs:` - 文档更改
- `style:` - 代码样式更改（格式化、分号等）
- `refactor:` - 代码重构
- `perf:` - 性能改进
- `test:` - 添加或更新测试
- `chore:` - 维护任务

### 贡献方向

- **性能**：优化渲染或模拟逻辑
- **功能**：添加新的单元格类型、行为或 UI 控件
- **文档**：改进 README、添加教程或代码注释
- **Bug 修复**：修复报告的问题
- **国际化**：添加新的语言翻译
- **测试**：添加单元或集成测试

### Pull Request 指南

- 包含清晰的更改描述
- 如适用，引用相关问题
- 确保代码构建无错误（`npm run build`）
- 如有 UI 更改，在不同浏览器中测试
- 如需要，更新文档
